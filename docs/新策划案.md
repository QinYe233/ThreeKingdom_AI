# 千秋弈 · 群雄逐鹿 v4.3 策划案

> **一句话定位**\
> 基于全图区块等时圈与战争迷雾，以**秩序与士气**衡量国力、以**信任与记忆**驱动外交、以**战局压力触发崩溃**。势力具备动态目标与情感记忆，地理纵深与专精倾向强化战略纵深。史官系统将数据翻译为历史叙事，一切兴亡皆在弈局之中。

## 一、核心设计理念

| 原则         | 说明                                   |
| ---------- | ------------------------------------ |
| **关系驱动**   | 势力行为优先受信任、记仇、记忆影响，而非单纯收益计算。          |
| **状态演化**   | 秩序与士气动态变化，影响经济与战斗，并通过战局压力触发崩溃。       |
| **目标动态切换** | 势力拥有扩张、防御、复仇、维稳、称帝等战略目标，由战局自动切换。     |
| **行为后果放大** | 完全开放背刺，后果由信任归零、记仇飙升、目标锁定等方式深刻改变后续博弈。 |
| **地理即战略**  | 区块人力、专精倾向、补给连通性决定战略价值；飞地与边陲带来独特治理挑战。 |
| **叙事涌现**   | 史官系统基于内部状态生成趋势分析与叙事文本，将数据翻译为历史故事。    |
| **可控失控**   | 通过聚焦目标、记忆长度限制、背刺频率冷却，防止系统陷入混乱螺旋。     |

## 二、地图与地理系统

### 2.1 地图规模与抽象

游戏采用高精度全图区块化地图，基于东汉十三州及周边地域真实地理数据抽象而成。全图共计 **361 个区块**，涵盖中原、江南、巴蜀、南中、凉州、幽州、辽东、交州及朝鲜半岛北部。

- 每个区块为唯一地理节点，拥有唯一名称、邻接关系列表、基础人力上限、当前人力池、归属势力、驻军数量、秩序、士气等属性。
- 初始剧本为公元 200 年 1 月。魏、蜀、吴分别控制中原、益州、江东的核心区域，其余区块为中立无主状态或由特殊中立势力控制。

| 区域类型    | 秩序/士气影响                     | 战局压力权重                |
| ------- | --------------------------- | --------------------- |
| **核心区** | 自然恢复速度正常；发展行动效果+10%         | 失守时战局压力 **+20 至 +30** |
| **羁縻区** | 基础秩序-10，秩序自然恢复速度减半；征兵效率-20% | 失守时战局压力 **+5 至 +10**  |

核心区争夺激烈，失守易引发连锁崩溃；羁縻区易攻难守，长期占领会拖累全国秩序。

### 2.3 补给线与飞地惩罚

- **补给线定义**：区块与本国首都之间存在**连续的本方控制区**路径，则视为**补给通畅**。
- **飞地状态**：若区块不与首都陆地连通，则该区块标记为**飞地**。
- **飞地惩罚**：
  - 每回合秩序额外 **-2**。
  - 新附状态下秩序额外 **-2**（与原新附惩罚叠加）。
  - 无法执行发展行动。
  - 驻军士气每回合 **-2**（最低降至 20）。
- **飞地战力惩罚**：飞地区块攻方战力 ×0.85，守方战力 ×0.9。

### 2.4 区块数据结构

```json
{
  "洛阳": {
    "neighbors": ["河东", "陈留", "宛城"],
    "base_manpower": 1400,
    "manpower_pool": 1400,
    "owner": "魏",
    "garrison": 800,
    "order": 80,
    "morale": 80,
    "last_recruit_round": -2,
    "develop_count": 0,
    "recently_conquered": false,
    "region_type": "core",
    "supply_connected": true,
    "geographic_trait": "fortress"
  }
}
```

字段说明：

- **neighbors**：相邻区块名称列表。
- **base\_manpower**：区块基础人力上限。
- **manpower\_pool**：当前可用人力池。
- **owner**：当前控制势力（魏/蜀/吴/中立/公孙度/士燮/南中/山越/凉州）。
- **garrison**：当前驻军数量。
- **order**：区块秩序值（0–100）。
- **morale**：区块士气值（0–100）。
- **last\_recruit\_round**：上次征兵回合数。
- **develop\_count**：已发展次数（0–3）。
- **recently\_conquered**：是否为新附区块。
- **region\_type**：`core`（核心区）或 `frontier`（羁縻区）。
- **supply\_connected**：是否与首都补给通畅。
- **geographic\_trait**：地理专精倾向，可为 `farming`、`trade`、`fortress` 或 `none`。

## 三、战争迷雾系统

迷雾随回合推移逐渐消除，分三阶段：

| 阶段       | 回合范围       | 迷雾规则                                                                                      |
| -------- | ---------- | ----------------------------------------------------------------------------------------- |
| **群雄割据** | 1–120 回合   | 完整迷雾：己方全见；邻接无主全见；邻接有主（非同盟）见归属、驻军估值、秩序士气区间；同盟邻接额外见基础人力与秩序；同盟非邻接见归属、驻军范围、秩序士气区间；其他仅见名称与邻接关系 |
| **天下渐明** | 121–240 回合 | 同盟非邻接可见归属与驻军范围；非同盟非邻接可见归属                                                                 |
| **混一之势** | 241 回合起    | 全图归属实时可见；非同盟驻军仍为保守估值（真实值×0.8），秩序士气可见文字区间                                                  |

阶段切换时触发史官叙事。非同盟驻军估值 = 真实值 × 0.8（随机浮动 ±10%）。

### 战斗预估胜率标签

可见敌方区块数据中提供预估胜率标签：

| 战力比     | 标签          | 描述        |
| ------- | ----------- | --------- |
| ≥ 1.5   | `"high"`    | 优势在我，胜算极大 |
| 1.0–1.5 | `"medium"`  | 旗鼓相当，胜负难料 |
| < 1.0   | `"low"`     | 敌强我弱，凶多吉少 |
| 信息不足    | `"unknown"` | 敌情不明，难以料算 |

战力比 = 攻方预估战力 / 守方预估战力（基于可见信息估算，武将修正与崩溃不计入）。

## 五、军事行动详解

### 5.1 移动兵力

- 目标为相邻己方区块，出发区块驻军减少，目标区块驻军增加。
- 单支部队每回合仅能主动移动一次，战斗撤退不受此限。

### 5.2 攻击

- 目标为相邻非己方区块，需指定出发区块、目标区块、派出兵力数。
- **最少出兵**：派出兵力 ≥ 出发区块驻军的 10%，且 ≥ 200。
- **袭扰例外**：若派出兵力 ≤ 500 且攻击敌方边境区块，可派最少 50 兵力。袭扰不触发占领，战后自动撤退，消耗 500 毫点。
- 可附带优先撤退路线。

### 5.3 征兵

- 条件：己方控制，黄金≥200，距上次征兵≥2回合，人力池≥10。
- 成本：200黄金。可征基数 = min(基础人力上限×30%，当前人力池×50%)，乘以秩序修正系数。
- 征兵后若人力池占比低于30%，全国秩序与士气各-2。
- **人力枯竭**：每次征兵使该区块基础人力永久减少征兵数量的10%。农垦专精可恢复。

### 5.4 解散兵力

- 消耗0毫点。条件：己方区块，驻军≥100。
- 每减少2驻军，人力池+1，不超过上限。每区块每回合限一次。

### 5.5 新附区块规则

- 归属变更后新附标记为真，无法征兵，人力自然恢复量减半。
- 若同时为飞地，每回合秩序额外-2。
- 占领方下一回合结束时自动清除新附标记。

## 六、经济与内政详解

### 6.1 发展区块

- 条件：己方控制，黄金≥400，发展次数<3。
- 成本：400黄金。效果：基础人力上限+200，人力池+100，目标秩序+3，全国秩序+1，发展次数+1。

### 6.2 发展专精与地理倾向联动

发展次数达3级后可选择专精方向，地理倾向区块效果增强20%。

| 专精名称 | 基础效果                   | 地理倾向联动                  |
| ---- | ---------------------- | ----------------------- |
| 农垦   | 基础人力上限+300，人力恢复比例提至20% | `farming` 区块：人力上限再+100  |
| 商贸   | 征税时该区块黄金贡献×1.5         | `trade` 区块：倍率提至 1.8     |
| 堡垒   | 防御系数下限提至1.05           | `fortress` 区块：下限提至 1.08 |

每个区块仅能选择一次专精。

### 6.3 征税

- 消耗500毫点。黄金收入 = 所有己方区块基础人力上限 × 动态系数 × 修正系数。
- 动态系数：`0.25 - (当前总控制区块数 / 1400)`，不低于 0.1。
- 修正系数：0.6 + min(全国秩序，区块秩序)/250。
- 全国秩序-5。秩序低于20的区块不产黄金但惩罚仍生效。

### 6.4 历史阶段对资源的修正（隐藏）

| 年份区间    | 全局修正           |
| ------- | -------------- |
| 200–208 | 无              |
| 208–220 | 全国人力恢复速度 -10%  |
| 220–234 | 全国黄金收入 -10%    |
| 234–260 | 全国秩序自然恢复 -1/回合 |
| 260–280 | 全国士气自然恢复 -1/回合 |

## 七、外交系统

### 7.1 关系数据结构

```json
{
  "魏-蜀": { "trust": 0.5, "grudge": 0.0 },
  "魏-吴": { "trust": 0.5, "grudge": 0.0 },
  "蜀-吴": { "trust": 0.5, "grudge": 0.0 }
}
```

### 7.2 关系变化规则

| 事件       | trust变化 | grudge变化 |
| -------- | ------- | -------- |
| 结盟成功     | +0.2    | 不变       |
| 每回合无互动   | -0.05   | -0.05    |
| 被攻击（宣战方） | 归零      | +0.5     |
| 达成停战     | +0.1    | 不变       |
| 背刺（攻击盟友） | 归零      | +0.7     |

### 7.3 发送外交消息

- 消耗：500 毫点。参数：`to`、`visibility`（`public`/`private`）、`content`。

### 7.4 即时对话机制

## 九、战斗与崩溃公式

### 9.1 战力计算

- 攻方战力 = 派出兵力 × (1 + 攻方全国士气/600) × 补给修正 × 兵力比修正
- 守方战力 = 驻军 × (1 + 守方全国士气/600) × 防御系数 × 补给修正
- 补给修正：飞地攻方×0.85，守方×0.9；通畅为1.0。
- 兵力比修正：攻方兵力 > 守方3倍时×1.1。
- 防御系数在0.95–1.15间随机。

### 9.2 胜负与战损

- 攻方大胜（战力比≥1.5）：守方全灭，区块易主，攻方损失为守方原驻军×0.8–1.2。
- 守方大胜（战力比≥1.5）：攻方损失为派出兵力×0–0.2，守方无损。
- 常规攻方胜：守方驻军归零，攻方损失同上，守方残兵溃逃15%–35%。
- 常规守方胜：攻方损失为派出兵力×0.7–1.0，守方损失为派出兵力×20%。
- 同归于尽：双方驻军≤0，区块变中立。

### 9.3 战局压力与士气崩溃

#### 战局压力

隐藏变量，单国0–100，每回合重置。

累积规则：

- 战败+15，大败+25
- 核心区失守+25（单回合上限40）
- 羁縻区失守+8
- 主力区块失守+20
- 被多方攻击+15
- 士气<40时每回合+5

#### 崩溃触发

- 战局压力≥60时触发士气崩溃，该方立即判负。
- 守方则驻军全灭，攻方则按守方大胜战损撤退，战后全国士气额外-5。
- 压力≥80必然触发。崩溃条件不写入决策提示。
